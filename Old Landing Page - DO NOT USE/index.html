04 — Red-Line Crisis Protocol

When Brand Health Score drops below 50, the swarm reacts automatically. Campaigns pause. Alerts fire. Recovery begins.

🚨 Auto-Trigger Sequence

1

Detection

red_line_monitor.py detects BHS < 50 (triggered by stale test data — BHS=24)

2

Dashboard → RED-LINE EMERGENCY

Status auto-set. Gauge turns red. Digest flags the crisis.

3

Kanban Alert Created

High-priority task auto-assigned to Marketing Manager.

4

Campaigns Auto-Paused

campaign_connector.py sends pause signal. Slack alert dispatched.

✅ Recovery Sequence

5

Autopsy

CI Agent delivers root-cause analysis. Identifies stale data as false positive.

6

Patch Deployed

Dev Agent fixes data pipeline → calculate_bhs.py reads from performance_metrics.json

7

BHS Restored to 93

Status reset to OPTIMIZED. Debrief logged. Incident closed.

8

Total Recovery Time: ~10 min
